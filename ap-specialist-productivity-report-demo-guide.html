1. Overview

The AP Specialist Productivity Report is a self-service web reporting tool built on .NET 8 and the EasyFile design system. It provides AP managers and supervisors real-time visibility into invoice processing performance across their team. The application connects to ECMToolbox Workflow databases, reads invoice and approval data, and presents it in an interactive, filterable dashboard with KPI summaries, rollup analytics, and CSV export.

Integration Point This application connects directly to the ECMToolbox Workflow SQL Server database. The Workflow Database connection is required; the AppEnhancer Database connection is optional and enables document index lookups. No changes to the ECMToolbox schema are required.

Key Capabilities

  • KPI Summary Strip — Five at-a-glance metrics: Total Invoices, Total Dollar Value, Unique Processors, Unique Vendors, and Average Processing Days. Cards update instantly when filters change.
  • Rollup Summary Tables — Three interactive summary panels: By AP Specialist, By Site Manager, and Aging Summary. Clicking any row filters the Invoice Detail table to that subset.
  • Invoice Detail Table — Full line-level detail with 15 columns including all approval steps, timing, and SLA color coding. Sortable and paginated at 25 rows per page.
  • SLA Color Coding — Green (0–3 days), Yellow (4–30 days), Red (31+ days) on all timing columns for instant visual identification of bottlenecks.
  • Flexible Date Filtering — Presets (Month to Date, Quarter to Date, Year to Date, Last Month, Last Quarter) or custom date ranges with four date basis options.
  • Dynamic Workflow Mapping — Configure field and queue mappings per workflow in the settings UI — no code changes required.
  • Windows Authentication — SSO via Negotiate (Kerberos/NTLM), with optional AD group-based access control.
  • CSV Export — One-click export of filtered data to Excel-compatible CSV.
  • Zero-Install Deployment — Self-contained .exe (~100 MB) requires no prerequisites on the host, or deploy to IIS for production environments.
Tip: Zero Prerequisites The self-contained executable bundles the .NET 8 runtime. No separate runtime installation is required on the server or workstation. Simply copy the .exe and run it to start serving the application.

2. Supported Scenarios

The following scenarios represent the primary use cases for the AP Specialist Productivity Report. Each can be demonstrated with a live instance connected to an ECMToolbox demo environment.

Scenario Description
AP Manager Daily Review Run a Month-to-Date report to see current invoice volumes, identify specialists with high open counts, and check aging buckets for overdue invoices. Click a specialist row in the rollup to drill into their specific invoices.
Site Manager SLA Monitoring Use the By Site Manager rollup to monitor approval turnaround times and SLA compliance (percentage of approvals completed within 3 days). Quickly identify managers with low SLA percentages.
Invoice Aging Analysis The Aging Summary table shows only in-progress invoices grouped by age bucket (0–3, 4–7, 8–14, 15–30, 31+ days) with dollar values. Click any age bucket to drill into those invoices.
Month-End Audit Run Last Month with AP Approval Date basis to see all invoices approved during the period. Export to CSV for reconciliation or audit documentation.
Cross-Workflow Reporting Organizations with multiple AP workflows can configure each independently and switch between them via the report’s workflow dropdown. Each workflow retains its own field and queue mappings.
Executive Summary Run Year-to-Date to see aggregate KPIs — total volume, dollar value, processor count, and average processing time — suitable for leadership presentations.

3. Configuration

The application has a built-in Settings page accessed via the gear icon or by navigating to /settings.html. Configuration covers three areas: database connections, workflow field & queue mappings, and access control.

3.1 Database Connections

Two side-by-side connection cards are displayed on the Settings page:

  • Workflow Database (Required) — SQL Server connection to the ECMT Workflow database containing workflows, work items, instances, queues, and users. Fields: Server, Database, Username, Password.
  • AppEnhancer Database (Optional) — SQL Server connection for document index data. Uses the same field layout as the Workflow Database card.

Click Test Connection on each card to verify connectivity. A successful test auto-saves the connection settings.

Security: Password Storage All passwords are encrypted using Windows DPAPI before being written to disk. Encrypted values are tied to the Windows account running the application and cannot be decrypted on a different machine or under a different account.

3.2 Workflow Configuration

After establishing the Workflow Database connection, configure each AP workflow:

  1. Select a workflow from the Workflow dropdown.
  2. Under Field Mappings, map each report column (Vendor Name, Invoice Number, Amount, Invoice Date, DocID, Workflow Status, Workflow Queue) to the corresponding workflow database field. Auto-detection pre-populates dropdowns by matching field names.
  3. Under Queue Role Mappings, assign each queue a role: Not Used, First Approval (Site Manager), or Second Approval (AP/Senior). Auto-detection matches queues by name patterns.
  4. Click Save Workflow Mapping. Repeat for each workflow in the organization.
Tip: Auto-Detection Field and queue mappings include an auto-detect step that pre-populates dropdowns based on field name matching. Review the pre-populated values before saving — most workflows will require little or no manual adjustment.

3.3 Access Control

Restrict report access to specific Active Directory groups:

  • Type 2 or more characters in the search field to query AD groups via LDAP.
  • Add groups to the Allowed AD Groups list. Users must be a member of at least one listed group to access the report.
  • Bootstrap mode: When no groups are configured, all authenticated Windows users have access. A yellow notice is displayed on the Settings page indicating this mode is active.
Authentication Mechanism The application uses Windows Authentication (Negotiate — Kerberos/NTLM). No username/password login prompt is shown to end users. The browser handles SSO automatically when the site is within the Local Intranet or Trusted Sites zone.

4. Usage & Testing (Demo Script)

Follow these steps to walk a prospect through the full reporting experience. Each step corresponds to a distinct UI area and key capability.

Step 1: Open the Application

Navigate to the application URL in the browser. Windows SSO authenticates the user automatically — no login prompt appears. The filter bar is displayed at the top of the page with controls for Workflow, Preset, Start Date, End Date, Date Basis, Processor, and Status, plus the Run Report and Export CSV buttons.

Tip: Browser Zone Configuration For seamless SSO, ensure the application URL is in the browser’s Local Intranet or Trusted Sites zone. If a credential prompt appears, add the site to the appropriate zone via Internet Options → Security.

Step 2: Run a Report

  1. Select a workflow from the Workflow dropdown (if multiple workflows are configured).
  2. Choose Year to Date or Month to Date from the Preset dropdown. Start and End Date fields populate automatically.
  3. Leave Date Basis as System Created Date.
  4. Click Run Report.

Step 3: Review KPI Summary

Five KPI cards appear immediately below the filter bar showing aggregate metrics for the selected period:

  • Total Invoices — Count of all invoices in the result set with the date range shown beneath.
  • Total Dollar Value — Sum of all invoice amounts for the period.
  • Unique Processors — Count of distinct AP specialists, labeled “AP specialists active.”
  • Unique Vendors — Count of distinct vendor names, labeled “Distinct vendor names.”
  • Avg Processing Days — Average days from system entry to final AP approval, labeled “Receipt to final approval.”
Demo Talking Point Demonstrate that KPI cards update instantly when any filter changes — no page reload required. Change the Processor or Status dropdown and show the cards recalculate immediately.

Step 4: Explore Rollup Tables

Three summary panels appear below the KPI strip:

By AP Specialist

Each processor is shown with their invoice count, average processing days (SLA color-coded), and number of open/in-progress invoices. Useful for identifying high-volume processors or individuals with elevated aging.

By Site Manager

Each manager is shown with assigned count, average approval days, pending count, and SLA percentage. The SLA percentage is the proportion of approvals completed within 3 days — color-coded green (100%), yellow (partial), red (low).

Aging Summary

In-progress invoices are grouped by age bucket: 0–3 days, 4–7 days, 8–14 days, 15–30 days, and 31+ days. Each row shows count, total dollar value, and count at the site manager step. Age bucket labels are color-coded to match SLA thresholds.

Tip: Click-to-Filter Drill-Down Click any row in a rollup table to filter the Invoice Detail table to that subset. A blue banner appears above the detail table indicating the active filter. Click Clear Filter in the banner to restore the full result set. Demonstrating this on the 31+ days aging bucket is a high-impact moment — it immediately isolates every overdue invoice for review.

Step 5: Review Invoice Detail

Scroll down to the Invoice Detail table. Key features to highlight:

  • 15 columns: AP Processor, Vendor Name, Invoice Date, Invoice #, Invoice Amount, System Entry, Site Mgr Approver, Site Mgr Date, Mgr Days, AP Approver, AP Approval Date, AP Days, Total Days, Comments, Status.
  • SLA color coding on the Mgr Days, AP Days, and Total Days columns.
  • Column sorting — click any column header to toggle ascending/descending order.
  • Pagination — 25 rows per page with navigation controls at the bottom of the table.
0–3 days — On track 4–30 days — Monitor 31+ days — Overdue

Step 6: Apply Filters

Use the filter controls in the top bar to narrow results further:

  • Processor dropdown — filter to a specific AP specialist.
  • Status dropdown — show only Completed or In Progress invoices.

All five KPI cards update immediately to reflect the filtered dataset.

Step 7: Export to CSV

Click Export CSV in the top-right of the filter bar. The browser downloads a file named AP_Productivity_Report_[start]_to_[end].csv. The exported file contains all data matching the active filters with UTF-8 BOM encoding for correct display in Microsoft Excel.

Step 8: Settings Demo (Admin)

Click the gear icon to navigate to the Settings page. Walk through each configuration section:

  1. Database Connections — Show the Workflow DB and AppEnhancer DB cards with the Test Connection button.
  2. Workflow Configuration — Select a workflow, show the Field Mappings dropdowns with auto-detected values, then demonstrate Queue Role assignment.
  3. Access Control — Show the AD group search field and the bootstrap mode notice banner.

5. Expected Results

After running a report, the following outputs and behaviors are expected:

KPI Summary Strip

Five summary cards display aggregate metrics for the selected date range and filters. Cards recalculate immediately when the Processor or Status filter changes without requiring a new Run Report action. The date range label beneath Total Invoices reflects the selected date window.

Rollup Tables

Three summary panels (By AP Specialist, By Site Manager, Aging Summary) appear below the KPI strip:

  • Clicking any data row filters the Invoice Detail table and displays a blue filter banner above the detail section.
  • Clicking Clear Filter in the banner restores the full unfiltered result set.
  • Only one rollup filter can be active at a time; selecting a different row replaces the previous filter.

Invoice Detail Table

A paginated table (25 rows per page) shows all invoice records matching active filters. SLA color coding on timing columns highlights items requiring attention. Completed invoices display a green status indicator; In Progress invoices display yellow.

CSV Export

A downloaded file named AP_Productivity_Report_[start]_to_[end].csv contains all filtered data. The file uses UTF-8 BOM encoding for correct character display when opened in Microsoft Excel. All 15 columns from the Invoice Detail table are included in the export.

Settings Persistence

All configuration settings persist across application restarts in an encrypted connections.json file stored alongside the application executable. Database passwords are encrypted using Windows DPAPI and are not readable as plain text.

Important: DPAPI Binding Because passwords are encrypted using Windows DPAPI, the connections.json file is bound to the Windows account that encrypted it. If the application is moved to a different server or run under a different Windows account, database passwords must be re-entered in the Settings page.
